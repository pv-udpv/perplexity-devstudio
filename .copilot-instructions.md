# Copilot Instructions for Perplexity DevStudio

## Project Context
This is a **browser userscript** built with Vite, React 18, and CodeMirror 6. It runs as an overlay on `perplexity.ai` OR as a raw file viewer on any domain.

## Architectural Principles

### 1. Zero Bundle Bloat (CRITICAL)
- **NEVER** import heavy libraries (monaco-editor, three.js, babel-standalone) directly.
- **ALWAYS** use `externalGlobals` in `vite.config.ts` or lazy-load via CDN.
- Use `react-json-view-lite` instead of full JSON viewers.
- Use `CodeMirror 6` (tree-shakable), NOT Monaco.

### 2. Mobile-First UX
- All UI must work on small touch screens (iOS Safari).
- Touch targets must be ≥44px.
- Toolbars must be sticky above the keyboard (use `position: sticky` or `visualViewport` API).
- Avoid `hover` states for critical functionality.

### 3. Security & Permissions
- **MINIMIZE** grants. Only use:
  - `GM.xmlHttpRequest` (for cross-origin AI/Gist calls)
  - `GM.setValue` / `GM.getValue` (for settings)
  - `unsafeWindow` (for intercepting `fetch`)
- **NEVER** expose full `GM` api to the page context.
- Sanitize all HTML output from AI.

### 4. Perplexity Integration
- Use the internal SSE endpoint `/rest/sse/perplexity_ask` when possible.
- Handle CORS via `GM.xmlHttpRequest` if running on non-perplexity domains.
- Respect the existing session cookies (do not prompt for login if already logged in).

## Coding Standards

### React Components
- Functional components with Hooks only.
- Strict TypeScript interfaces for all props.
- Inline styles for critical layout (to avoid CSS collisions), CSS modules for themes.
- Use `React.memo` for high-frequency updates (like network logs).

### CodeMirror 6
- Use `EditorView.theme` for styling (scoped).
- Use `showPanel` for toolbars (don't hack DOM directly).
- Use `StateField` for state management.

### Error Handling
- Wrap AI calls in try/catch and show user-friendly errors.
- Graceful degradation if `GM` API is missing.

## Workflow
1. Check `package.json` for dependencies before importing.
2. If adding a large dep, configure `externalGlobals`.
3. Update `README.md` if adding new features.
